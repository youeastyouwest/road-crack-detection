# 全流程道路裂缝智能检测系统 — 技术选型方案

---

> 本文档确定项目全部技术选型，作为团队开发的技术规范依据。

---

## 一、技术选型总览

| 层次 | 技术选型 | 版本要求 | 选择理由 |
|------|---------|---------|---------|
| **前端框架** | Vue 3 + TypeScript | Vue 3.4+ / TS 5+ | 组合式 API 灵活、TypeScript 类型安全、生态成熟 |
| **前端构建** | Vite | 5.x | 开发热更新极快、构建速度快 |
| **前端 UI 库** | Element Plus | 2.x | Vue 3 官方推荐、组件丰富、中文文档友好 |
| **前端图表** | ECharts | 5.x | 性能优异、支持实时数据流、社区活跃 |
| **前端通信** | Axios + WebSocket (原生) | — | Axios 封装 HTTP、WebSocket 实现实时推送 |
| **后端框架** | Spring Boot | 3.x | Java 生态主流框架、稳定成熟、社区资源丰富 |
| **后端构建** | Maven | 3.9+ | 依赖管理成熟、广泛使用 |
| **JDK** | JDK 17 (LTS) | 17+ | Spring Boot 3.x 要求、长期支持 |
| **实时通信** | Spring WebSocket + STOMP | — | Spring 原生支持、与后端无缝集成 |
| **ORM 框架** | MyBatis-Plus | 3.5+ | 简化 SQL 编写、代码生成器提升效率 |
| **权限安全** | Spring Security + JWT | — | 业界标准方案、RBAC 支持完善 |
| **数据库** | MySQL 8.0 | 8.0+ | 稳定可靠、开源免费、团队熟悉 |
| **缓存** | Redis | 7.x | 高性能缓存、支持消息队列 |
| **算法框架** | PyTorch 2.x | 2.0+ | 研究社区主流、ONNX 导出支持好 |
| **模型推理** | ONNX Runtime | 1.15+ | 跨平台、高性能、支持 GPU 加速 |
| **模型检测** | YOLOv8 | — | 实时目标检测 SOTA、训练部署一体 |
| **模型分割** | U-Net (改进版) | — | 裂缝像素级分割、适合小目标 |
| **部署** | Docker + Docker Compose | — | 环境一致性、一键部署 |
| **反向代理** | Nginx | 1.24+ | 静态资源托管、API 反向代理 |
| **项目管理** | Maven (多模块) | — | 前后端可统一管理 |

## 二、各端技术栈详解

### 2.1 前端（客户端工程师）

```
┌────────────────────────────────────┐
│           前端技术栈                │
├────────────────────────────────────┤
│  Vue 3 (Composition API)           │
│  + TypeScript                      │
│  + Vite (构建工具)                  │
│  + Element Plus (UI 组件库)        │
│  + ECharts (数据可视化)            │
│  + Axios (HTTP 通信)               │
│  + WebSocket (实时推送)            │
│  + Pinia (状态管理)                │
│  + Vue Router (路由)               │
└────────────────────────────────────┘
```

### 2.2 后端（服务端工程师）

```
┌────────────────────────────────────┐
│           后端技术栈                │
├────────────────────────────────────┤
│  Spring Boot 3.x                   │
│  + JDK 17                          │
│  + Maven (多模块构建)               │
│  + Spring WebSocket (实时)         │
│  + Spring Security + JWT (鉴权)    │
│  + MyBatis-Plus (数据库 ORM)       │
│  + MySQL 8.0 (数据存储)            │
│  + Redis (缓存/队列)               │
│  + Knife4j (API 文档)              │

```

**Maven 模块结构：**

```
server/
├── pom.xml                          # 父 POM
├── crack-common/                    # 公共模块（工具类、常量）
├── crack-api/                       # API 接口定义（DTO、VO）
├── crack-service/                   # 业务逻辑层
├── crack-dao/                       # 数据访问层（MyBatis Mapper）
├── crack-websocket/                 # WebSocket 实时通信模块
├── crack-report/                    # 报告生成模块
└── crack-bootstrap/                 # 启动入口
```

### 2.3 数据处理（数据处理工程师）

```
┌────────────────────────────────────┐
│        数据处理/算法技术栈           │
├────────────────────────────────────┤
│  Python 3.9+                       │
│  + PyTorch 2.x (深度学习框架)      │
│  + YOLOv8 (裂缝目标检测)           │
│  + U-Net (裂缝语义分割)            │
│  + ONNX Runtime (模型推理加速)     │
│  + OpenCV (图像处理)               │
│  + NumPy / Pillow (数据操作)       │
│  + Albumentations (数据增强)       │
│  + Flask (轻量推理 API)            │
└────────────────────────────────────┘
```

**集成方式：**

```
                    推理服务调用链
                    
  Spring Boot (后端)                    Python (算法推理)
┌──────────────────┐     HTTP/gRPC    ┌──────────────────┐
│  业务逻辑层        │ ──────────────→ │  Flask 推理服务   │
│  (接收视频帧)      │                 │  (加载 ONNX 模型) │
│                   │ ←────────────── │                   │
│  返回检测结果      │   返回 JSON     │  YOLOv8 + U-Net   │
│  →推送给前端       │                 │  →裂缝坐标+掩码   │
└──────────────────┘                 └──────────────────┘
```

## 五、技术选型决策记录

| 决策点 | 方案 A | 方案 B | 最终选择 | 理由 |
|-------|--------|--------|---------|------|
| 前端框架 | Vue 3 | React 18 | **Vue 3** | 团队熟悉度更高、上手快 |
| UI 组件库 | Element Plus | Ant Design Vue | **Element Plus** | 与 Vue 3 契合度更高 |
| 后端框架 | Spring Boot | FastAPI | **Spring Boot** | 项目要求 Spring Boot |
| 数据库 | MySQL | PostgreSQL | **MySQL** | 团队更熟悉、运维简单 |
| ORM | MyBatis-Plus | JPA | **MyBatis-Plus** | SQL 可控性强、学习成本低 |
| 模型框架 | PyTorch | TensorFlow | **PyTorch** | 研究社区主流、ONNX 导出方便 |
| 推理部署 | ONNX Runtime | TensorRT | **ONNX Runtime** | 通用性强、支持多后端 |
| 实时通信 | WebSocket | SSE | **WebSocket** | 双向通信、适合视频帧推送 |
| 容器化 | Docker Compose | K8s | **Docker Compose** | 项目规模小、简单够用 |

## 六、环境搭建速查

### 前端

```bash
# 创建项目
npm create vue@latest client
cd client
npm install
npm install element-plus echarts axios pinia vue-router@4
npm install -D unplugin-auto-import unplugin-vue-components

# 开发启动
npm run dev          # 默认 http://localhost:5173
```

### 后端

```bash
# 前提：JDK 17 + Maven 3.9
# 使用 Spring Initializr 生成项目骨架
# 或在 IDEA 中直接创建 Spring Boot 项目

# 添加依赖（pom.xml）
# spring-boot-starter-web
# spring-boot-starter-websocket
# spring-boot-starter-security
# mybatis-plus-spring-boot3-starter
# mysql-connector-j
# spring-boot-starter-data-redis
# lombok
# knife4j-openapi3-jakarta-spring-boot-starter
# jjwt-api / jjwt-impl / jjwt-jackson

# 启动
mvn clean install -DskipTests
java -jar crack-bootstrap/target/crack-bootstrap.jar
```

### Python 算法服务

```bash
# 创建虚拟环境
cd data-processing
python -m venv venv
venv\Scripts\activate   # Windows

# 安装依赖
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118
pip install ultralytics  # YOLOv8
pip install opencv-python numpy pillow onnx onnxruntime-gpu
pip install flask flask-cors  # 推理API服务
pip install albumentations scikit-learn tqdm

# 验证 GPU
python -c "import torch; print(torch.cuda.is_available())"
```
