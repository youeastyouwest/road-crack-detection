# 全流程道路裂缝智能检测系统 — 项目文档

---

## 1 引言

### 1.1 项目背景

随着我国交通基础设施建设的快速发展，道路总里程已位居世界前列。然而，道路在日常使用过程中，受车辆荷载、温度变化、水分侵蚀等自然和人为因素影响，不可避免地会产生裂缝等病害。传统道路裂缝检测主要依赖人工巡检方式，即由检测人员徒步或驾车沿路检查并记录裂缝位置、类型和严重程度。这种检测方式存在以下突出问题：

- **效率低下**：人工巡检速度慢，需要大量人力资源，尤其对于高速公路和偏远路段，检测周期长、覆盖范围有限。
- **安全性差**：检测人员在车流密集路段作业时面临较大的交通安全风险。
- **主观性强**：检测结果高度依赖个人经验，不同检测人员对同一裂缝的判定标准不一致，缺乏统一的量化指标。
- **数据管理困难**：人工记录的数据难以系统化存储和追溯，历史对比分析困难，无法为道路养护决策提供科学依据。

近年来，无人机技术、计算机视觉和深度学习技术的快速发展，为道路病害检测带来了全新的解决方案。无人机搭载高清摄像头可以快速获取道路影像数据，深度学习模型能够自动识别和分类裂缝，结合前后端分离架构可以实现从数据采集、实时分析到报告生成的全流程闭环管理。

### 1.2 项目意义

本项目的实施具有以下重要价值：

**（1）提升检测效率**
通过无人机视频流实时分析与自动化裂缝识别，能够大幅缩短道路检测周期。相比人工巡检，系统将检测效率提升数倍以上，实现对道路网络的快速普查和重点路段的实时监控。

**（2）保障人员安全**
以无人机代替人工进行道路巡检，从根本上消除了检测人员在交通环境中作业的安全风险，体现了"科技兴安"的理念。

**（3）提高检测精度**
借助深度学习模型，系统能够识别微细裂缝（宽度达到毫米级），并提供量化的裂缝长度、宽度、面积等指标，避免人工检测的主观差异，实现检测标准的统一化和精确化。

**（4）科学辅助决策**
系统根据道路异常严重程度，自动触发 AI Agent 生成详细检测报告，为道路养护部门提供有数据支撑的科学养护方案，实现从"被动维修"到"主动预防"的转变。

**（5）数据全生命周期管理**
支持历史数据存储、查询、对比分析和趋势预测，帮助养护部门掌握道路病害的发展规律，优化养护资源的配置。

### 1.3 项目内容

本项目开发一套全流程道路裂缝智能检测系统，主要功能包括：

- **无人机视频流接入与实时分析**：支持 RTMP/RTSP 视频流的实时接入，自动抽帧并进行裂缝检测，检测结果实时展示在仪表盘上。
- **自动化裂缝识别与量化评估**：基于深度学习目标检测和语义分割模型，自动识别图像中的裂缝区域，并计算裂缝的长度、宽度、面积、密度等量化指标。
- **历史数据管理与趋势分析**：存储每次检测的任务记录、原始图像和分析结果，支持按道路、时间等维度进行查询、对比分析，生成道路病害发展趋势图。
- **多级告警与报告生成**：根据裂缝严重程度（轻微、中等、严重）自动分级告警，并由 AI Agent 自动生成详细的检测报告和养护建议，格式支持 PDF/Word。
- **前后端分离部署**：前端采用 React/Vue 框架提供可视化交互界面，后端采用 FastAPI 提供 RESTful API 和 WebSocket 实时通信服务。

项目最终成果包括：完整的检测软件系统、训练好的裂缝检测模型、技术文档和实地部署验证报告。

### 3.2 主要功能业务流程

#### 3.2.1 视频流检测流程

```
[无人机/视频源] → [视频流接入] → [实时抽帧] → [裂缝检测模型推理]
                                                    ↓
                                              [生成检测结果]
                                                    ↓
         ┌────────────────────────────────────────────┐
         ↓                    ↓                        ↓
    [仪表盘实时展示]   [告警/通知]              [存储到历史数据库]
         ↓                    ↓                        ↓
    [查看量化指标]      [严重→报告Agent]         [后续对比分析]
```

**流程说明：**
1. 用户接入无人机视频流（实时RTMP）或上传视频文件
2. 系统自动对视频流进行帧提取（可配置抽帧频率）
3. 每帧图像送入裂缝检测模型进行推理
4. 检测结果（裂缝位置、宽度、长度等）实时推送到前端仪表盘
5. 根据裂缝严重程度触发告警，严重时自动调用 Agent 生成报告
6. 所有检测数据自动存储，支持后续回放与分析

#### 3.2.2 报告生成流程

```
[检测任务完成/告警触发] → [收集检测数据] → [裂缝严重度分级]
                                                    ↓
                                              [轻微 → 忽略/记录]
                                              [中等 → 简要报告]
                                              [严重 → 详细报告]
                                                    ↓
              [AI Agent 分析数据] → [生成报告文本+图表] → [人工审核(可选)]
                                                    ↓
                                        [导出 PDF/Word / 在线预览]
```

### 3.3 功能需求说明

#### 3.3.1 功能模块 1：用户管理

| 角色 | 说明 | 权限范围 |

| 检测员 | 日常检测操作人员 | 视频流管理、实时监控、历史查询 |
| 审核员 | 报告审核与决策人员 | 查看检测结果、审核报告、导出文档 |

**用例描述：**

| 用例名称 | 用户登录 |
|---------|---------|
| 用例概述 | 用户通过用户名和密码登录系统，获取对应角色权限 |
| 前置条件 | 用户已在系统中注册 |
| 后置条件 | 登录成功，进入系统主界面并按角色展示对应功能菜单 |

| 用例名称 | 视频流接入 |
|---------|-----------|
| 用例概述 | 用户配置视频流源地址和参数，启动实时检测任务 |
| 前置条件 | 用户已登录，系统服务正常运行 |
| 后置条件 | 视频流成功接入，开始实时检测并将结果推送至前端 |

#### 3.3.2 功能模块 2：视频流接入与管理

**功能描述：** 支持多种视频源的接入与配置。

**详细功能点：**
- **实时流接入**：支持 RTMP、RTSP、HTTP-FLV 等实时视频流协议，配置流地址即可接入
- **视频文件上传**：支持上传 MP4、AVI、MOV 等常见格式的视频文件进行离线分析
- **参数配置**：可配置抽帧频率（每秒/每2秒等）、检测区域（ROI）设置
- **状态监控**：实时显示视频流连接状态、帧率、丢帧情况

#### 3.3.3 功能模块 3：实时检测与展示

**功能描述：** 对视频流进行实时裂缝检测，并将结果直观展示。

**详细功能点：**
- **实时检测**：对视频帧进行裂缝检测，标注裂缝区域（Bounding Box + Mask）
- **量化指标**：实时计算裂缝长度（像素/毫米）、最大宽度、平均宽度、裂缝面积、裂缝密度
- **仪表盘展示**：以仪表盘形式展示当前检测数据，包括：检测帧数、裂缝数量、当前严重等级、实时指标曲线
- **告警触发**：当检测到严重裂缝时，前端弹窗告警、声音提示

#### 3.3.4 功能模块 4：历史数据管理与分析

**功能描述：** 存储和查询历史检测数据，支持多维度分析和可视化。

**详细功能点：**
- **检测记录管理**：按时间、道路段、严重等级等维度查询历史检测记录
- **详情查看**：查看单次检测的原始图像、标注图像、量化指标、检测报告
- **趋势分析**：以折线图、柱状图等展示某条道路的裂缝发展趋势（裂缝数量/宽度随时间变化）
- **道路健康评估**：综合历史数据对道路健康状况进行评分和分级

#### 3.3.5 功能模块 5：报告管理与导出

**功能描述：** 基于 AI Agent 自动生成检测报告，支持多级告警和报告导出。

**详细功能点：**
- **多级告警机制**：
  - 轻微裂缝（宽度 < 0.2mm）：记录归档，定期汇总
  - 中等裂缝（宽度 0.2~0.5mm）：生成简要报告，发送通知
  - 严重裂缝（宽度 > 0.5mm）：立即告警，自动生成详细报告
- **AI Agent 报告生成**：Agent 自动分析检测数据，撰写报告内容包括：检测概况、裂缝分布统计、严重路段标记、养护建议
- **报告格式**：支持在线预览、导出 PDF / Word 文档
- **专家审核**：生成的报告支持人工编辑和审核后再发布

#### 3.3.6 功能模块 6：系统管理

**功能描述：** 系统配置与运维管理。

**详细功能点：**
- **模型管理**：上传/更新/回滚裂缝检测模型，查看模型版本信息
- **系统配置**：检测参数默认值、告警阈值、报告模板配置
- **日志管理**：系统操作日志、检测日志、错误日志的查看与导出
- **审计跟踪**：记录关键操作的时间、用户、操作内容

### 3.4 非功能需求说明

#### 3.4.1 性能需求

| 需求项 | 指标要求 | 验证方法 |
|--------|---------|---------|
| 检测帧率 | ≥ 15 FPS（实时视频流） | 使用性能监控工具测试连续5分钟平均帧率 |
| 单帧检测延迟 | ≤ 200ms（从帧到达模型到返回结果） | 统计1000帧检测延迟的P95值 |
| 视频接入延迟 | ≤ 3秒（从视频源推流到前端显示） | 使用秒表测量推流到显示的延迟 |
| 历史查询响应 | ≤ 2秒（10万条记录内按条件查询） | 使用测试脚本模拟并发查询 |
| 报告生成时间 | ≤ 30秒（单次检测任务全量报告） | 测试不同数据量下的报告生成时间 |
| 系统并发用户 | ≥ 10个用户同时在线使用 | 使用压测工具模拟10个用户并发操作 |
| 系统可用性 | ≥ 99.5%（月累计） | 监控系统运行时间，统计故障时长 |

#### 3.4.2 精度需求

| 需求项 | 指标要求 |
|--------|---------|
| 裂缝检测精确率（Precision） | ≥ 90% |
| 裂缝检测召回率（Recall） | ≥ 85% |
| 裂缝宽度测量误差 | ≤ ±0.1mm（标定后） |
| 误报率 | ≤ 5%（非裂缝区域被标记为裂缝的比例） |

#### 3.4.3 安全性需求

- **身份认证**：用户登录需验证，密码采用加密存储（bcrypt 或同类算法）
- **权限控制**：基于角色的访问控制（RBAC），不同角色只能访问授权的功能
- **通信安全**：前后端通信使用 HTTPS/WSS 协议，防止数据被截取
- **数据安全**：检测数据和报告定期备份，支持数据恢复
- **接口鉴权**：所有 API 接口使用 JWT Token 鉴权，防止未授权访问

#### 3.4.4 可用性需求

- **界面友好**：采用中文界面，操作直观，新用户能在 30 分钟内上手基本操作
- **操作引导**：关键操作有提示和确认，防止误操作
- **错误提示**：系统错误时给出明确的错误信息和处理建议，而非无响应或堆栈信息
- **响应式设计**：支持不同分辨率的显示器，适配大屏展示场景

#### 3.4.5 可扩展性需求

- **模块化设计**：系统采用前后端分离架构，各功能模块解耦，支持独立升级
- **模型热加载**：模型更新无需重启服务，支持不停机更换检测模型
- **接口标准化**：API 接口遵循 RESTful 规范，便于第三方系统集成
- **横向扩展**：后端服务支持多实例部署，通过负载均衡实现横向扩展
